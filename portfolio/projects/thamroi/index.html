Overview

ThamRoi is a collaborative social platform built with TypeScript, Next.js (frontend) and Node.js / Express / Prisma (backend), developed by the G08SE25 Kikkaksaranungkukuk team following Agile/Scrum methodology across multiple sprints (Aug 2025 – Nov 2025). I contributed full-stack features spanning authentication, search, profile integration, admin moderation, and deployment fixes.

Frontend Development (TypeScript / Next.js / SCSS)

  • Designed and implemented the Group Searching page UI from scratch, including search input, filter controls, and results display components.
  • Integrated the Group Search page with the backend REST API, handling query parameters and dynamic data rendering.
  • Connected the Profile View page to the backend, enabling live user data display.
  • Built a comprehensive Admin Reports Dashboard with full backend integration:
    • Implemented JWT-based authentication and authorization guard, with unauthorized access fallback to a custom NotFound component.
    • Normalized and decoded report/tag data from varying backend response shapes for consistent UI rendering.
    • Implemented dynamic report tag fetching from backend, replacing hardcoded values.
    • Added search, filter, and pagination controls with loading state management.
    • Built an IssueDetailDialog modal component for detailed report viewing with resolve/unresolve actions.
  • Implemented the User Consent / Terms of Service page, enforcing sequential reading of Terms of Service and Privacy Policy before allowing agreement.
  • Updated legal pages (Terms of Service, Privacy Policy) to reflect application rebranding to "ThamRoi".
  • Identified and resolved a deployment environment configuration bug where the frontend was defaulting to localhost:8080 instead of the .env-configured backend URL.

Backend Development (TypeScript / Node.js / Express / Prisma / PostgreSQL)

  • Added API routing to support the Group Search frontend, enabling search and filter queries.
  • Diagnosed and fixed group search routing and filter logic bugs to restore correct search behavior.
  • Extended the backend to support the Profile View page, adapting existing endpoints and data models.
  • Developed the Blog Post Filter API endpoint (/api/v2/blog), supporting multi-parameter filtering via query strings.
  • Implemented a fetch interests API endpoint, refactoring interests from an enum to a relational database table for flexibility.
  • Refactored the entire Report system to support the Admin Dashboard:
    • Replaced a multi-tag array system with a single reason string mapped to a report_tag relation (auto-created with "OTHER" fallback).
    • Added a public endpoint to fetch all available report reasons/tags.
    • Implemented is_resolved boolean field across report creation, update, and filtering with validation middleware.
    • Updated type definitions and validation middleware throughout the report module.
  • Created database seed scripts (admin_seed.ts, report_seed.ts) for test data setup and admin account provisioning.
